# Workspace

This folder is the **agent workspace** — a dedicated area for the Rayliant AI agent to store intermediate reasoning artefacts, session state, generated outputs and operational logs.

## Purpose

The workspace acts as the agent's scratchpad. During a session the agent may write intermediate results, drafts, and context files here before they are committed to the database or delivered to the user.

## Planned Contents

| Subfolder / File | Description |
|---|---|
| `sessions/` | Per-session context files capturing the agent's current reasoning state |
| `drafts/` | Emails, reports and other content generated by the agent awaiting review or send |
| `reports/` | Completed sales reports and pipeline summaries produced by the agent |
| `logs/` | Operational logs for agent actions, tool calls and skill executions |
| `temp/` | Short-lived temporary files; contents may be cleared between runs |

## Usage Guidelines

- **Do not commit sensitive data** (e.g. prospect PII, credentials) stored in the workspace.
- Add `workspace/sessions/`, `workspace/drafts/`, `workspace/logs/` and `workspace/temp/` to `.gitignore` for local development.
- Completed artefacts (e.g. approved reports) should be archived to the [database](../database/README.md) or an external store rather than left in the workspace indefinitely.
